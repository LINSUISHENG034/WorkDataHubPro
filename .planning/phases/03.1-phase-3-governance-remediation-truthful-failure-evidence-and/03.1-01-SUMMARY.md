## Decisions Made

- **Shared helper placement in `runtime.py`:** Keeps checkpoint-to-payload logic centralized while letting each slice runner own the actual `baseline_paths`, `legacy_payloads`, and `pro_payloads` maps it passes in
- **Semantic of legacy_result vs pro_result in compatibility case:** `legacy_result["rows"]` carries the mismatching baseline (the evidence of what failed); `pro_result["rows"]` carries the correct intermediate payload (what the slice actually produced) - both are needed for adjudication review
- **`_bootstrap_intermediate_baselines` return value:** Was returning `None`; corrected to return `outcome.intermediate_payloads` so tests can reference the correct baseline payloads for assertion
