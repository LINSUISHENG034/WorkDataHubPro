<objective>
Fix truthful failure evidence compatibility-case payloads to use in-memory intermediate_payloads captured as a local variable, not file-system reads that can be corrupted by the test.

Purpose: When a test corrupts a baseline file after bootstrap, the compatibility_case must still reference the correct baseline (from the in-memory intermediate_payloads), not the corrupted file content.
Output: All three slice runners compute intermediate_payloads as a local variable before the compatibility-case block, use it for legacy_payloads, and pass it to SliceRunOutcome.
</objective>

<context>
The issue is structural: in each slice runner, `intermediate_payloads` is currently built **inline** inside the `SliceRunOutcome(...)` constructor call (at the END of the function). But `legacy_payloads` for the compatibility case is needed earlier (before `SliceRunOutcome` exists).

**Current broken structure (example from annuity_performance_slice):**
```python
# Lines 493-499: legacy_payloads built from FILES (can be corrupted)
legacy_payloads = {
    "fact_processing": expected_fact_processing,   # from _load_replay_baseline
    ...
}

# Lines 507-514: compatibility case built using legacy_payloads
failure_locator, legacy_result, pro_result = build_failure_compatibility_case_payload(
    primary_failure.checkpoint_name,
    baseline_paths=baseline_paths,
    legacy_payloads=legacy_payloads,  # uses file data
    pro_payloads=pro_payloads_map,
)

# Line 568+: SliceRunOutcome created AT END of function
return SliceRunOutcome(
    ...
    intermediate_payloads={  # <- inline dict, not captured earlier
        "reference_derivation": reference_derivation_payload,
        "fact_processing": _build_fact_payload(resolved_facts),
        ...
    },
)
```

**Correct structure:**
```python
# BEFORE compatibility-case block: capture intermediate_payloads as local variable
intermediate_payloads = {
    "reference_derivation": reference_derivation_payload,
    "fact_processing": _build_fact_payload(resolved_facts),
    "identity_resolution": _build_identity_payload(resolution_results),
    "contract_state": contract_state.rows,
    "source_intake": _build_observed_source_intake_contract(records),
}

# In compatibility-case block: use the LOCAL variable
legacy_payloads = intermediate_payloads  # NOT reading from files
failure_locator, legacy_result, pro_result = build_failure_compatibility_case_payload(
    primary_failure.checkpoint_name,
    baseline_paths=baseline_paths,
    legacy_payloads=legacy_payloads,  # uses in-memory intermediate_payloads
    pro_payloads=pro_payloads_map,
)

# At SliceRunOutcome: reuse the same variable
return SliceRunOutcome(
    ...
    intermediate_payloads=intermediate_payloads,  # same dict, no duplication
)
```

This ensures the compatibility case uses the in-memory intermediate_payloads dict (captured during bootstrap) rather than reading potentially-corrupted file contents.
</context>
