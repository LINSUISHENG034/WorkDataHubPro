## Warnings

### WR-01: Bootstrap baseline corruption risk on intermediate checkpoint failure

**File:** `tests/replay/test_annuity_performance_slice.py:96-126`
**Also affected:** `tests/replay/test_annual_award_slice.py:133-163`, `tests/replay/test_annual_loss_slice.py:188-218`

**Issue:** The `_bootstrap_intermediate_baselines` helper writes empty `[]` baseline files before running the slice, then overwrites them with actual results. If the first `run_*_slice` call (line 107/144/199) raises an exception, the baseline files on disk remain as empty arrays. Subsequent test runs or other tests sharing the same replay_root could silently use corrupted baselines, causing confusing test failures that do not reproduce locally.

**Fix:**
```python
def _bootstrap_intermediate_baselines(workbook_path: Path, replay_root: Path) -> dict[str, list[dict[str, object]]]:
    # Pre-compute baselines in memory first, then write atomically
    checkpoint_names = (
        "reference_derivation",
        "fact_processing",
        "identity_resolution",
        "contract_state",
    )
    # Write empty baselines first so the files exist
    for checkpoint_name in checkpoint_names:
        (replay_root / f"legacy_{checkpoint_name}_2026_03.json").write_text(
            "[]",
            encoding="utf-8",
        )
    
    # Now run the slice to get actual baselines
    outcome = run_annuity_performance_slice(  # or run_annual_award_slice / run_annual_loss_slice
        workbook=workbook_path,
        period="2026-03",
        replay_root=replay_root,
    )
    
    # Write actual baselines; if this fails, the in-memory outcome is preserved for diagnosis
    for checkpoint_name in checkpoint_names:
        (replay_root / f"legacy_{checkpoint_name}_2026_03.json").write_text(
            json.dumps(
                outcome.intermediate_payloads[checkpoint_name],
                indent=2,
                ensure_ascii=False,
            ),
            encoding="utf-8",
        )
    return outcome.intermediate_payloads
```

Alternatively, use a temporary directory for bootstrap that is distinct from the final `replay_root` passed to the actual test assertions.

### IN-01: Mutable default argument in function signature

**File:** `src/work_data_hub_pro/apps/orchestration/replay/diagnostics.py:111`

**Issue:** `find_comparison_run_root` uses `registry=REPLAY_DOMAINS` as a default argument. While REPLAY_DOMAINS is a module-level constant that is not mutated, this pattern is a common Python antipattern. If the registry were ever mutated (e.g., `REPLAY_DOMAINS[key] = value`), the mutation would persist across calls.

**Fix:**
```python
def find_comparison_run_root(
    comparison_run_id: str,
    registry=None,  # None instead of REPLAY_DOMAINS
) -> tuple[str, Path]:
    if registry is None:
        registry = REPLAY_DOMAINS
    ...
```

### IN-02: Same pattern in etl_cli main module

**File:** `src/work_data_hub_pro/apps/etl_cli/main.py:126`

**Issue:** `replay_diagnose` command uses `registry=REPLAY_DOMAINS` as a default argument. Same concern as IN-01.

**Fix:** Same pattern - default to `None` and resolve to the module constant inside the function.

### IN-03: LEGACY_TEMP_PREFIX constructed with string concatenation

**File:** `tests/replay/test_annual_loss_slice.py:18`

**Issue:** `_LEGACY_TEMP_PREFIX = "TE" + "MP-"` produces `"TEMP-"`. This is likely intentional (distinguishing from `_OPAQUE_TEMP_ID` which starts with `"IN"`), but the concatenation makes the intent less obvious. Consider:
```python
_LEGACY_TEMP_PREFIX = "TEMP-"  # more explicit
# Or if the split is intentional for some reason, add a comment explaining why
```
