<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| accepted baseline assets -> replay gate runtime | Silent fallback when a baseline is missing would convert a parity gate into a self-compare no-op. |
| compatibility diff generation -> human and machine review | Overcounted duplicate-row mismatches would inflate perceived divergence severity. |
| bootstrap command -> accepted reference assets | A hidden or implicit bootstrap path would let runtime execution mint baselines without review. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02.1-01 | T | reference-derivation baseline loading | mitigate | Raise immediately when `legacy_reference_derivation_<period>.json` is absent and cover the message contract in tests. |
| T-02.1-02 | I | duplicate-row diff reporting | mitigate | Fix `_build_diff(...)` to subtract duplicate counts one occurrence at a time and add duplicate-row regression tests. |
| T-02.1-03 | R | baseline bootstrap provenance | mitigate | Keep baseline creation in a standalone script with explicit CLI flags and no automatic replay fallback. |
</threat_model>
