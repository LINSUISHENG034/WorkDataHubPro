<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| runtime payloads -> persisted evidence JSON | Sensitive business identifiers must not cross the persistence boundary in plaintext. |
| redaction helper -> audit/diagnostic anchors | Masking must not remove `comparison_run_id`, `batch_id`, `record_id`, `anchor_row_no`, or rule metadata needed for lookup and adjudication. |
| policy file -> writer behavior | If field rules drift between writers, evidence safety becomes inconsistent and unverifiable. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | I | persisted evidence packages | mitigate | Route all governed writes through one redaction helper and one governed policy file with an explicit `***REDACTED***` token. |
| T-04-05 | T | audit-anchor integrity | mitigate | Preserve `comparison_run_id`, `batch_id`, `record_id`, `anchor_row_no`, `stage_id`, `rule_id`, and `checkpoint_name` exactly and assert them in integration tests. |
| T-04-06 | R | policy drift between artifact types | mitigate | Freeze policy keys and required literal field lists in contract tests so future changes cannot silently diverge. |
</threat_model>
