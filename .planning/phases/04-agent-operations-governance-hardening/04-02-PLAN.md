<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| runtime payloads -> persisted evidence JSON | Sensitive business identifiers must not cross the persistence boundary in plaintext. |
| redaction helper -> audit/diagnostic anchors | Masking must not remove `comparison_run_id`, `batch_id`, `record_id`, `anchor_row_no`, `trace_path`, `origin_row_nos`, or `parent_record_ids`. |
| policy file -> writer behavior | If field rules drift between writers or are reloaded inconsistently, evidence safety becomes noisy and unverifiable. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | I | persisted evidence packages | mitigate | Route all governed writes through one redaction helper and one governed policy file with an explicit `***REDACTED***` token. |
| T-04-06 | T | lookup-critical anchor integrity | mitigate | Preserve `trace_path`, `origin_row_nos`, `parent_record_ids`, `comparison_run_id`, `batch_id`, `record_id`, and `anchor_row_no` exactly and assert them in the lookup regression test. |
| T-04-07 | R | policy drift and repeated IO | mitigate | Load `config/policies/evidence_redaction.json` once per `FileEvidenceIndex` instance and freeze the required policy keys and list values in contract tests. |
</threat_model>
