<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller intent -> file-backed compatibility case transitions | Invalid or under-specified transitions must not silently mutate adjudication records. |
| file-backed case record -> audit trail | Missing owner or closure proof turns a case into an unaccountable state snapshot. |
| CLI surface -> service lifecycle rules | The command layer must not bypass service validation or write partial lifecycle fields directly. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | R | compatibility case ownership | mitigate | Require explicit owner and record it in `decision_owner`, `closed_by`, and `decision_history` for every non-pending transition. |
| T-04-08 | T | invalid status mutation | mitigate | Enforce the exact allowed statuses and required note/evidence fields in `transition_case(...)` and cover them with integration tests. |
| T-04-09 | R | closure-proof gaps | mitigate | Make `close_case(...)` require at least one closure-evidence path and persist it alongside timestamps and actor fields. |
</threat_model>
