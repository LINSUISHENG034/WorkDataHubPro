| caller intent -> file-backed compatibility case transitions | Invalid or under-specified transitions must not silently mutate adjudication records. |
| authoritative case file -> per-run mirror | The dual persistence paths must not drift after create, transition, or close operations. |
| CLI surface -> service lifecycle rules | The command layer must not bypass service validation or write partial lifecycle fields directly. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-04-08 | R | compatibility case ownership | mitigate | Require explicit owner and record it in `decision_owner`, `closed_by`, and `decision_history` for every non-pending transition. |
| T-04-09 | T | invalid status mutation | mitigate | Enforce the exact `allowed_transitions` map and the failure codes `missing_owner`, `empty_resolution_note`, `empty_closure_evidence`, and `illegal_transition` in service and CLI tests. |
| T-04-10 | R | dual-store persistence drift | mitigate | Treat `compatibility_cases/<case_id>.json` as authoritative and mirror the same serialized payload to `comparison_runs/<comparison_run_id>/compatibility-case.json` after every successful mutation, with integration tests asserting both files match. |
</threat_model>
