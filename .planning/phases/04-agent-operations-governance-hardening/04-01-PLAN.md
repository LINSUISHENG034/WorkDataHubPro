<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| untrusted `comparison_run_id` / `record_id` input -> evidence package lookup | CLI input must not escape governed replay roots or resolve arbitrary files. |
| comparison-run package -> row-level diagnostics output | Missing or incomplete lineage package fields would make lookup silently drop source-stage evidence. |
| failed/warning run evidence -> operator diagnostics | The lookup contract must still return useful anchors and case references when the run did not pass cleanly. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | T | replay lookup CLI input | mitigate | Reuse existing `comparison_run_id` validation rules, require either `record_id` or `anchor_row_no`, and test the exact option contract. |
| T-04-02 | I | lineage-impact package | mitigate | Persist exact `record_id`, `batch_id`, `anchor_row_no`, `origin_row_nos`, `parent_record_ids`, and `trace_path` fields and assert them in integration tests for passed and failed runs. |
| T-04-03 | R | failed-run diagnostics | mitigate | Include `checkpoint_statuses` and `compatibility_case_id` in lookup output so failed runs stay auditable instead of becoming partial black boxes. |
</threat_model>
