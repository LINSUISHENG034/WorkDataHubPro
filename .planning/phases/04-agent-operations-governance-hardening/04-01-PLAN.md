<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| untrusted `comparison_run_id` / selector input -> evidence package lookup | CLI input must not escape governed replay roots or resolve arbitrary files. |
| comparison-run package -> row-level diagnostics output | Missing or malformed lineage package data must not silently downgrade to partial output. |
| failed-run evidence -> operator diagnostics | Missing trace artifacts on failed runs must remain explicit and queryable rather than hiding row anchors. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | T | replay lookup CLI input | mitigate | Reuse existing `comparison_run_id` validation rules, require exactly one selector, and freeze the exact error codes `missing_selector`, `conflicting_selectors`, and `invalid_comparison_run_id` in contract tests. |
| T-04-02 | I | lineage-impact package | mitigate | Persist exact `record_id`, `batch_id`, `anchor_row_no`, `origin_row_nos`, `parent_record_ids`, `trace_path`, and `artifact_gaps` fields and assert them in integration tests for passed and failed runs. |
| T-04-03 | R | malformed lineage package data | mitigate | Fail closed with exact errors `missing_lineage_package` and `malformed_lineage_package` instead of returning partial silent output. |
| T-04-04 | R | failed-run diagnostics | mitigate | Return retained `checkpoint_statuses` and explicit `trace_missing` gaps so failed runs stay auditable instead of becoming partial black boxes. |
</threat_model>
