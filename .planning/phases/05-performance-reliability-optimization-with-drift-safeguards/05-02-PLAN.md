<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| trace-event store snapshot -> lazy lookup index | Read-path index construction must preserve the current `_events` snapshot, then lazily refresh on the next read after later appends, without altering append behavior. |
| trace-event store -> explainability consumers | Faster lookup must not reorder or drop trace events that replay diagnostics depend on. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03 | T | lazy trace read-path indexing | mitigate | Keep index construction on the read path only, lazily rebuild on the next read after snapshot changes, and prove that freshness and performance behavior in benchmark and contract coverage without adding write-path responsibilities. |
| T-05-04 | R | explainability ordering drift | mitigate | Freeze ascending `event_seq` ordering in contract tests and run replay explainability tests before completion. |
