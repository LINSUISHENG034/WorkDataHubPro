Current fragile behavior:
- `load_publication_policy(...)` does `json.loads(...)` then `payload[domain]`
- `build_publication_plan(...)` does `policy.targets[target_name]`
- `PublicationService.execute(...)` appends `PublicationResult(..., success=True)` for every processed bundle

Locked implementation constraints:
- use Pydantic v2 model
- exact typed error family under `PublicationPolicyError`
- load-time fail-fast for file/parse/schema/domain errors
- lookup-time fail-fast for unknown target
- first bundle execution failure raises typed error and stops later bundles
</interfaces>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| publication policy file -> validated runtime contract | File absence, malformed JSON, or structural drift must fail before execution starts. |
| publication target selection -> publication plan | An unknown target must not collapse into an unstructured `KeyError`. |
| bundle execution -> publication results | A failing write must not be reported as a successful publication and must stop later bundles. |

| T-05-06 | T | bundle execution truthfulness | mitigate | Raise `PublicationExecutionError` on first failing bundle, stop later bundles, and prove that behavior in integration tests. |
</threat_model>
