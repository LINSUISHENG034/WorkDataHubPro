<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process environment -> postgres adapter | untrusted DSN/schema input controls connection target and schema selection |
| adapter -> database | SQL write path crosses into persistent state and must fail closed on schema drift |
| committed schema snapshot -> runtime validation | repository assets define the allowed table shape for a live database |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | I | env-driven connection config | mitigate | read DSN only from `WDH_PG_DSN` or `DATABASE_URL` per D-07 and never commit secrets to config files |
| T-06-05 | T | `postgres_tables.py` SQL execution | mitigate | use parameterized SQL for row values and restrict identifiers to the four policy-locked target names from D-02 |
| T-06-06 | T | schema validation gate | mitigate | compare `information_schema.columns` against committed `*.schema.json` and raise `PostgresSchemaMismatchError` on first mismatch per D-09 |
| T-06-07 | D | adapter connection/write failures | mitigate | wrap connection and write failures in typed PG errors so callers fail fast instead of hanging on ambiguous driver exceptions |
</threat_model>
