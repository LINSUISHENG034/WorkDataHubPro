|----------|-------------|
| annuity slice -> shared replay runtime | The shared runtime must not absorb annuity-specific business or fixture logic. |
| setup/config failures -> run report | Invalid setup must raise typed exceptions instead of being reported as a governed mismatch. |
| shared runtime -> evidence package output | Report/evidence path generation must stay aligned with the accepted Phase 2 package schema. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | T | shared replay runtime boundary | mitigate | Keep only D-02 primitives in `runtime.py`, leave manifest paths, fixture loading, and annuity publication/projection wiring explicit in the slice file, and assert adoption in replay tests. |
| T-03-08 | R | setup-failure reporting | mitigate | Route baseline/config/validator failures through `ReplaySetupError` and verify they raise before checkpoint evaluation starts. |
| T-03-09 | I | run-report evidence linkage | mitigate | Build `primary_failure` and evidence-path metadata from accepted checkpoint/gate outputs and keep package writing on the existing Phase 2 helper path. |
</threat_model>
