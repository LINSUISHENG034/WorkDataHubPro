<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| raw company name -> generated temp identity | Unredacted names must not leak into public identifiers or evidence references. |
| shell/CI environment -> HMAC temp-id generation | Missing or unstable salt configuration would break determinism and could expose a weak fallback policy. |
| identity service -> downstream replay payloads | Placeholder names must remain null-like values instead of becoming a shared fake customer id. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | I | temp-id generation | mitigate | Generate opaque `IN...` ids with HMAC-SHA1 + Base32, keep raw names only in `evidence_details`, and assert no raw-name substring appears in `company_id` or `evidence_refs`. |
| T-03-05 | T | temp-id policy drift | mitigate | Put prefix and salt-env settings in one governed `config/releases/temp_identity_policy.json` file and reject per-domain/per-run overrides. |
| T-03-06 | D | placeholder-name handling | mitigate | Return `None` for placeholder inputs and cover it in integration tests so runs do not collapse distinct missing-name cases into one fake identifier. |
</threat_model>
