<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `comparison_run_id` input -> evidence package lookup | Untrusted CLI/agent input must not escape the governed replay roots or load arbitrary files. |
| replay runtime -> error/report contract | Setup failures and completed mismatches must stay distinct so agents do not treat invalid runs as governed parity differences. |
| domain discovery metadata -> future CLI surface | Hidden or relative-path metadata would make agent execution depend on shell context instead of governed paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | T | comparison-run diagnostics lookup | mitigate | Reject path separators in `comparison_run_id`, scan only registry-declared replay roots, and cover missing-run behavior in `tests/contracts/test_replay_diagnose_contracts.py`. |
| T-03-02 | R | replay failure taxonomy | mitigate | Freeze a typed setup-exception hierarchy and separate it from `ReplayRunReport` so invalid setup cannot masquerade as a compatibility result. |
| T-03-03 | I | registry metadata and replay-root resolution | mitigate | Store repo-root-anchored `replay_root`, `runbook_path`, `release_path`, and `domain_config_path` in `REPLAY_DOMAINS` instead of relying on CWD-relative defaults. |
</threat_model>
